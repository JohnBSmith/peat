Constraints

Code generation

The code generator should produce polymorphic code.

Reason 1: Allows for fast compile times.

Reason 2: Is necessary for dynamic loading.

Dynamic loading should be supported.

Reason: Allows quick startup of software and unlimited functionality.

The code generator should produce bytecode.

Reason 1: Allows for cheap portability.

Reason 2: The code generator may be exchanged at a later point of time.

The bytecode interpreter should be a stack machine.

Reason 1: Allows a cheap implementation.

Reason 2: May be exchanged at a later point of time.
